## 5. 铁律 / 约定（违反会出错或返工）
- 【待填：环境调用规矩。例：绘图用 base env，栅格用 X env，二者不可同进程（DLL 冲突）】
- 【待填：数据规矩。例：离散数据重采样必须 near/mode，绝不双线性/三次】
- 【待填：口径/单位规矩。例：组分数据以 53µm 粒径法为主干，密度法/氧化法不混入主干回归】
- 【待填：命名规矩。例：中文图含 ²/µ 用 mathtext（$^2$/$\mu$），SimHei 缺字形会显方框】
- 【待填：跨数据集去重规则。例：按 DOI + 坐标(±0.01°) + 数值近似 去重】
- **代码归位**：脚本进 `scripts/`，不散落根目录。每新增一个脚本，同步更新 `scripts/README.md`（写清这个脚本干嘛、输入、输出、谁写的）。根目录只放入口文件和薄指针。
- **收工规矩**：每个 agent 结束时更新本文件 §3/§4 + 写/更新 `STATUS.md`（handoff 落盘：现状/下一步/风险/关键路径）。保证下个 agent **不依赖对话历史也能续上**。

## 5.5 路径约定（重要：避免云同步爆炸 / 多 agent 撞车）
项目根 `【待填：绝对路径】` 在【待填：是否云同步目录，如 OneDrive】里【会/不会】自动同步。大量/大体积/中间文件**绝不能**直接写进项目根，否则同步卡死。

| 目录 | 用途 |
|---|---|
| `<项目根>\` | 小产物（KB–几 MB 的 CSV、PNG、脚本、md、最终少量栅格）|
| `F:\xxx_scratch\` | 大体积/中间件（GB 级 RF 中间栅格、整包，绝不进项目）|
| `F:\xxx_tmp\` | 通用临时 |

- **判据**：单个 >50MB 或一次产 >十几个文件 → 进 scratch；小文件进项目。
- 脚本里**输出路径写成常量置顶**（如 `SCRATCH=r'F:\xxx_scratch'`、`PROJ_OUT=r'...\output'`），方便核查与改盘。
- 原始数据源（Zotero/手动下载源数据）**只读，勿写勿删**。
