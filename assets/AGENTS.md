## 5. 铁律 / 约定（违反会出错或返工）
- **不可混用的维度**（硬约束）【待填：列出本领域绝对不能混用的维度。例：53µm 粒径法 ≠ 密度法/氧化法；g/kg ≠ Mg/ha（差 1000 倍）；Albers 投影 ≠ WGS84 坐标；表层(0-20cm) ≠ 深层(20-40cm) 不可合并回归；不同时期的数据不可混为同一期】
- 【待填：环境调用规矩。例：绘图用 base env，栅格用 X env，二者不可同进程（DLL 冲突）】
- 【待填：数据规矩。例：离散数据重采样必须 near/mode，绝不双线性/三次】
- 【待填：口径/单位规矩。例：组分数据以 53µm 粒径法为主干，密度法/氧化法不混入主干回归】
- 【待填：命名规矩。例：中文图含 ²/µ 用 mathtext（$^2$/$\mu$），SimHei 缺字形会显方框】
- 【待填：跨数据集去重规则。例：按 DOI + 坐标(±0.01°) + 数值近似 去重】
- **代码归位**：脚本进 `scripts/`，不散落根目录。每新增一个脚本，同步更新 `scripts/README.md`（写清这个脚本干嘛、输入、输出、谁写的）。根目录只放入口文件和薄指针。
- **收工规矩**：每个 agent 结束时更新本文件 §3（累计快照，带日期）+ §4 看板 + 写/更新 `STATUS.md`（**增量 handoff**：本轮做了什么/动了哪些文件/踩了什么坑，不重复 §3）。收工时跑 `python scripts/check_handoff.py` 自检——验证 §3 日期新鲜、TL;DR 已填、STATUS.md 非模板、薄指针存在。全过才算交接合格。
- **独立复核阀门（关键任务）**：影响论文结论或下游管线的输出，必须由主控/另一 agent 独立抽验关键数字（守恒/边界/格数/量级/各文档一致性）通过后才算完成。"自检全过"不构成通过判据。独立复核通过前，产物标记为"待验，下游不得启用"。
- **关键数字单一来源**：同一数字（均值/百分比/面积/统计量等）只在 STATUS.md 或 AGENTS.md §3 一处写定，别处引用不复述。收工前核对所有文档间数字一致性——同一数字在不同文档差 1% 以上即视为 bug，必须对齐才能交。
- **坏产物退役隔离**：发现某 agent 产物错了，立即做四件事：① 目录/文件名加 `_DEPRECATED` 后缀或移入 `_作废/` 子目录；② 在 §5（铁律节）顶部用红字 `> ⚠️ 【作废】<路径> <原因>` 标注（别标在 §3——§3 是累计快照，坏产物标记应和铁律/规范同在 §5）；③ 更新 §4 看板状态为 `[x]` + 标注"作废"；④ 更新 STATUS.md 反映作废。绝不只靠记忆说"那个别用"。

## 5.5 路径约定（重要：避免云同步爆炸 / 多 agent 撞车）
项目根 `【待填：绝对路径】` 在【待填：是否云同步目录，如 OneDrive】里【会/不会】自动同步。大量/大体积/中间文件**绝不能**直接写进项目根，否则同步卡死。

| 目录 | 用途 |
|---|---|
| `<项目根>\` | 小产物（KB–几 MB 的 CSV、PNG、脚本、md、最终少量栅格）|
| `【待填：工作盘如 D:\scratch 或 /mnt/scratch】\` | 大体积/中间件（GB 级 RF 中间栅格、整包，绝不进项目）|
| `【待填：临时目录】\` | 通用临时 |

- **判据**：单个 >50MB 或一次产 >十几个文件 → 进 scratch；小文件进项目。
- 脚本里**输出路径写成常量置顶**（如 `SCRATCH=r'F:\xxx_scratch'`、`PROJ_OUT=r'...\output'`），方便核查与改盘。
- 原始数据源（Zotero/手动下载源数据）**只读，勿写勿删**。
